The payoff

What you get back

~4,500lines removable

Across ~35 confirmed-dead files (plus 2 to verify). That's less code to read, secure, test, and onboard new developers into — with zero behaviour change, because the app never imports any of it.

How to remove it safelyDo it on a branch: delete the Tier-1 files, then boot the app and run a smoke test. Because nothing imports them, an import error would surface immediately if something was missed. Relocate the Tier-2 scripts to scripts/. Resolve the two Tier-3 files by hand. Commit in small, labelled batches so anything can be reverted cleanly.
Done when
  • The 15 dead app modules and 12 dead chunkers are deleted; the app boots and passes a smoke test.
  • The patch leftovers (patch_main, session_auth_patch) are gone, so there's one authoritative version of each file.
  • One-off scripts live in scripts/, and the duplicated update_user_fields_from_excel.py is down to a single copy.
  • The 2 verify-first files are confirmed and either kept or removed — no lingering "maybe dead" modules.
  • A .gitignore / lint rule discourages new stray scripts and *_patch.py files from creeping back.
