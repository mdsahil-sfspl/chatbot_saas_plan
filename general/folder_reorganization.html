Folder reorganization — from flat god-folders to clear subpackages

Two directories do almost everything: core/ holds 29 files mixing six unrelated concerns, and api/ holds 20 mixing routers, services, and leftovers. Grouping them into subfolders makes the codebase far easier to navigate — with no behaviour change.

How to do it without breaking anything

Behaviour-preserving move, in safe steps

This touches imports everywhere — respect thatMoving files changes every import that points at them. Done carelessly it breaks the app; done in small IDE-assisted steps it's routine. The rules below keep it safe.
The safe sequence
  • 1. Delete dead code first (the ~4,500 lines) so you move less and file away nothing wasted.
  • 2. One subpackage at a time — create core/rag/, move its files, let the IDE auto-update imports, boot + smoke-test, commit. Repeat for llm/, config/, etc.
  • 3. Add __init__.py to every new package so imports resolve.
  • 4. Keep main.py where it is so the Docker/uvicorn entrypoint doesn't change; update include_router import paths as routers move.
  • 5. Grep for string imports and dynamic loads (e.g. provider-by-name) — those don't auto-update and must be fixed by hand.
  • 6. Commit in small, labelled batches — one folder per commit — so any move can be reverted cleanly.
Best time to do thisRight after the dead-code cleanup and ideally before multi-tenancy (Task 12) — reorganizing is far cheaper before big new features land on top of the current layout.
