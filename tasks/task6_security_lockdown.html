Task 5 — proxy & security lockdown

Your security headers are genuinely strong — better than most. But two things are wide open: the app trusts proxy headers from anyone, and CORS falls back to a wildcard with credentials. Here's exactly what to tighten.

Where you are today

What your code does right now

strongSecurity headers — CSP, HSTS, X-Frame-Options, nosniff, COOP/COEP/CORP all set in SecurityHeadersMiddleware.
goodCookies mostly hardened — HttpOnly, SameSite, and a COOKIE_SECURE switch via the _cookie_set helper.
openProxy trust — ProxyHeadersMiddleware(trusted_hosts="*") believes X-Forwarded-For from anyone.
openCORS fallback — if ALLOWED_ORIGINS is unset it becomes ["*"] with allow_credentials=True.
minorOne cookie (pub_session_id, main.py:553) is set by hand and misses the Secure flag.
minorCSP allows 'unsafe-inline'; Permissions-Policy allows microphone=*.
Why the proxy line is the priority With trusted_hosts="*", any client can send a forged X-Forwarded-For header and pretend to be any IP. That silently defeats your IP-keyed rate limits (Task 2's login 3/min) and poisons every log and audit trail with fake source IPs.
The risk, visually

Who does the app trust?

A reverse proxy rewrites the client's real IP into X-Forwarded-For. The app must trust that header only when it comes from your own load balancer — never from the public internet.

What to do · in order

The four moves

Two critical (5a, 5b), two hardening (5c, 5d). All in api/main.py and api/session_auth.py.

5a

Lock proxy trust to your load balancer

critical · ~2 hrs
api/main.py:128.env (TRUSTED_PROXY_IPS)
  1. Find your proxy/LB internal IP (Azure App Gateway / nginx). That's the only host allowed to set forwarded headers.
  2. Replace the wildcard with that IP list from an env var. 
    # api/main.py — was: trusted_hosts="*"
TRUSTED = [p.strip() for p in os.getenv("TRUSTED_PROXY_IPS", "").split(",") if p.strip()]
app.add_middleware(ProxyHeadersMiddleware, trusted_hosts=TRUSTED or "127.0.0.1")
  3. Verify — send a request with a bogus X-Forwarded-For directly (not via the LB); the app should log your real IP, not the fake one.
Do this before trusting Task 2Rate limits are only as honest as the IP behind them. Fixing this makes your login limit actually protect against brute force.
5b

Make CORS fail closed

critical · ~2 hrs
api/main.py:276–286
  1. Remove the wildcard fallback. Today an empty ALLOWED_ORIGINS becomes ["*"] — and with allow_credentials=True, that lets any website make logged-in requests to your API.
  2. Require explicit origins and narrow methods/headers to what you use. 
    # api/main.py
origins = [o.strip() for o in os.getenv("ALLOWED_ORIGINS", "").split(",") if o.strip()]
if not origins:
    raise RuntimeError("ALLOWED_ORIGINS must be set")   # fail closed, never "*"
app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE", "OPTIONS"],
    allow_headers=["Authorization", "Content-Type"],
)
  3. Set ALLOWED_ORIGINS in every environment (your domain + local dev). Prod must never boot without it.
The ruleallow_origins=["*"] and allow_credentials=True must never appear together — it's the classic cross-site data-theft misconfig.
5c

Make cookies consistently secure

hardening · ~2 hrs
api/main.py:553api/session_auth.py (COOKIE_SECURE)
  1. Fix the hand-rolled cookie. The pub_session_id cookie at line 553 sets HttpOnly; SameSite=Lax but no Secure. Route it through your existing helper so it inherits Secure: 
    # api/main.py:553 — instead of a raw Set-Cookie string
_cookie_set(resp, "pub_session_id", pub_sid, max_age=86400)
  2. Force COOKIE_SECURE=true in production so no session cookie is ever sent over plain HTTP.
  3. Confirm SameSite is Lax (or Strict where flows allow) for all auth cookies.
5d

Polish the (already good) headers + review sweep

hardening · ~half a day
api/main.py (SecurityHeadersMiddleware)requirements.txt
  1. Add WebSocket to CSP. You use /ws/chat; make sure connect-src allows your wss:// origin so the socket isn't blocked in strict browsers.
  2. Tighten Permissions-Policy — change microphone=* to microphone=(self) so only your own origin can use the mic.
  3. Plan away 'unsafe-inline' in CSP over time (move inline scripts/styles to files or nonces). Not urgent, but it's the weakest part of an otherwise strong CSP.
  4. Quick review sweep: run pip-audit on prod deps, confirm no secrets in the repo (your earlier scan was clean), ensure debug/reload is off in prod, and restrict SSH/admin ports to known IPs.

Definition of done

Task 5 is finished when…

Done when
  • 5a — proxy trust is limited to the load-balancer IP; a forged X-Forwarded-For from outside is ignored (verified).
  • 5b — CORS never falls back to "*"; the app refuses to start without ALLOWED_ORIGINS; methods/headers narrowed.
  • 5c — every auth cookie (including pub_session_id) is HttpOnly + Secure + SameSite; COOKIE_SECURE=true in prod.
  • 5d — CSP allows your wss:// origin; microphone scoped to self; pip-audit clean; debug off; admin ports restricted.
  • The strong headers you already have are left intact — this task tightens, it doesn't rebuild.
